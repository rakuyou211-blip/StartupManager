# StartupManager 🍂

**PCの電源を入れてから、使えるようになるまでが長い。**
その「待たされてるあいだ」に、裏で勝手に立ち上がっているアプリたちがいます。

音楽プレーヤーの更新チェッカー、買ったときから入ってたメーカー製の常駐、いつインストールしたか覚えてないツール…。そういう「起動と同時に走り出すもの」を1つの画面に全部集めて、**止める・戻す・消す**を落ち着いて選べるようにする道具です。

「詳しくないから、どれを止めていいか分からない」——そこがいちばんこわいところだと思うので、**止めても元に戻せる**（Windowsなら）ことと、**消す前に必ずバックアップを取る**ことを、いちばん大事に作りました。

これは「落葉いの道具箱」のひとつです。小さな困りごとに、手作りの道具を。
まだ生まれたてで、私（落葉い）ひとりで作っています。荒いところは正直に書きます。

## 🍂 落ち葉の約束

この道具箱の全部に共通する約束です。

- **集めない・送らない** — テレメトリ0。使用状況も何も送信しません。動作はすべてPCの中だけで完結します。
- **登録なし** — アカウントもメールアドレスもいりません。
- **中身は全部読める** — ソース全公開（MITライセンス）。PowerShellとbashのスクリプトそのものなので、何をしているか自分の目で（詳しい人に頼んででも）確認できます。
- **無料・広告なし。**
- **ネットがなくても動く** — Windowsの標準コマンドと、macOSの標準コマンドだけを使っています。

## こんなときに

- 電源を入れてから、まともに操作できるまでが長い。
- タスクバーの右下（通知領域）に、いつの間にかアイコンが増えている。
- アンインストールしたはずのアプリが、まだ起動時に何かしようとして「見つかりません」的なエラーを出す。
- 起動時に立ち上がるものを見直したいけど、設定があちこちに散らばっていて分からない。

「機能がすごい」より先に、**この困りごと、心当たりありますか？** というところから作っています。

## できること

### 散らばった「起動するもの」を1画面に集める
Windowsの起動項目って、実はいろんな場所に分かれて登録されています。

- レジストリの Run キー
- スタートアップフォルダ
- タスクスケジューラ
- ストアアプリ（UWP）の自動起動

これを全部かき集めて、ひとつのリストで見られるようにしました。

### 止める・戻す・消す
- **止める（無効化）** — 一時的にオフにします。Windowsの「タスクマネージャーが使っている仕組み（StartupApproved）」に沿って止めるので、**あとから同じ画面でオンに戻せます。**
- **消す（削除）** — 項目そのものを取り除きます。**消す前に自動でバックアップを取り**、あとから復元できます（詳しくは下の「安全のはなし」）。

### 見分ける手がかりを出す
- 発行元・ファイルの場所・**デジタル署名の状態**
- いま実行中かどうか
- **ゴースト検出** — 実行ファイルがもう存在しない（アンインストール済みなのに登録だけ残った）項目を赤くして教えます。

### そのほか
- 実行ファイルを**ドラッグ＆ドロップ**してスタートアップに追加
- リストを**CSV**で書き出し
- キーボード操作：F5（更新）／Ctrl+A（全選択）／Delete（無効化）／Enter（詳細）／F1（ヘルプ）

## 使い方

### Windows（GUI）
1. ダウンロードした一式を、どこか分かりやすいフォルダに置きます（インストール不要）。
2. **`StartupManager.bat`** をダブルクリックします。
3. **「ユーザーアカウント制御（UAC）」の確認が出ます** → 「はい」。

> **なぜ管理者権限（UAC）を求めるの？**
> スタートアップ項目は、あなた個人の設定だけでなく「このPCを使う全員」に効く場所（システム側のレジストリなど）にも登録されています。そこまで正しく読み取って、止めたり戻したりするには管理者の権限が必要です。`StartupManager.bat` は、確実に動くように**最初から管理者として起動する**作りになっています。
> （中で何をしているかは `StartupManager.bat` と `StartupManager.ps1` を開けばそのまま読めます。）

### macOS（β・コマンドライン）
macOS版は **まだβ** です。画面のあるGUIではなく、ターミナルで番号を選んで操作する形です。正直まだ荒く、実機での検証を集めている段階です。

1. ダウンロードした一式を置きます。
2. ターミナルで、そのフォルダに移動して実行権限を付けます：
   ```
   chmod +x StartupManager.command
   ```
3. `./StartupManager.command` を実行、またはFinderでダブルクリック。

対象：ユーザーLaunchAgents／全体LaunchAgents／LaunchDaemons／ログイン項目。

> **macOSの正直な注意：ログイン項目は「消したら元に戻せません」。**
> LaunchAgents などはバックアップから復元できますが、**「ログイン項目」だけは名前しか記録できず、削除すると自動復元ができません。** 消す前に、本当に不要か確かめてください。

## はじめての人へ（警告が出たら）

初めて起動したとき、Windowsなら「**WindowsによってPCが保護されました**（SmartScreen）」、macOSなら「**開けません**」、あるいはウイルス対策ソフトが警告を出すことがあります。

**びっくりさせてすみません。でも、これは異常ではありません。**

理由はシンプルで、**私が個人開発者で、有料の「デジタル署名（コード署名証明書）」をまだ持っていないから**です。署名がないソフトは、中身が安全かどうかに関係なく、OSが「作者を確認できません」として一律に警告を出します。悪いことをしているから出るのではなく、「知らない人が作ったものですよ」という注意書きです。

**なぜ安全と言えるか：** このソフトは中身が全部ソースコード（PowerShell / bash のテキスト）で公開されています。隠れて何かをする余地がなく、何をするかは誰でも読めます。不安なら、詳しい人に `StartupManager.ps1` を見てもらってください。

### それでも進めるには

**Windows（SmartScreen）**
1. 「WindowsによってPCが保護されました」の画面で「**詳細情報**」をクリック。
2. 下に出る「**実行**」ボタンを押す。
3. 続いてUAC（管理者の確認）が出たら「はい」。← これは前述のとおり、起動項目を正しく扱うために必要です。

**macOS（Gatekeeper）**
- ファイルを**右クリック →「開く」**→ もう一度「開く」。（ダブルクリックだと拒否されても、右クリックからなら許可の選択肢が出ます。）
- それでもダメなときは、ターミナルで属性を外す方法があります：
  ```
  xattr -d com.apple.quarantine StartupManager.command
  ```

ここで諦めてほしくないので、正直に書きました。警告の理由がわかれば、こわくないはずです。

## 安全のはなし（迷ったら、まず「止める」）

一番伝えたいこと：**分からないものは、いきなり消さないでください。**

- 発行元を見ても正体が分からない項目は、**まず「止める」だけ**にしてください。止めるのは一時的なオフなので、もし必要なものだった場合でも、同じ画面から**戻せます**（Windowsの場合）。しばらく使ってみて問題がなければ、そのとき初めて消すか決めれば大丈夫です。
- **消す（削除）前には、自動でバックアップを取ります。** Windowsでは、消した項目は `Backups\` フォルダに保存され、あとからワンクリックで復元できます。
- **自己テスト（セルフテスト）モード**があります。実際のスタートアップ項目には一切手を触れずに、動作だけを確かめられます。試すのがこわい人は、まずこれで感触を掴んでください。
- v1.8.0 で、この**復元・バックアップまわりのバグをまとめて修正**しました（バックアップが作られずに戻せなくなる不具合、タスクの復元先パスがずれる不具合など）。生まれたてなので、まだ想定外があるかもしれません。おかしな挙動を見つけたら教えてください。

> それでも、これはあなたのPCのシステム設定（レジストリ／タスクスケジューラ／launchd）に触れる道具です。バックアップと復元は用意していますが、**最終的にはご自身の判断でお使いください。** 迷ったら「止める」から。
