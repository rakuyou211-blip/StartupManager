# Changelog

## v1.9.0 (2026-07-13)

### 追加
- **「サードパーティのみ」表示フィルタ**: Microsoft製・Windows標準の起動項目を隠して、
  掃除対象になりやすいサードパーティ製だけを一覧表示。起動を軽くしたいときに便利。
  - 判定: 発行元がMicrosoft / 実行ファイルがWindowsフォルダ配下 / MicrosoftのUWP(発行元ハッシュ 8wekyb3d8bbwe)を「標準側」として除外
  - フィルタ中はステータスバーに「Microsoft/標準 N 件を非表示」と表示
  - 設定は `settings.json` に記憶
- セルフテストにサードパーティ判定の検証を追加(Microsoft発行元/Windows配下/UWP/第三者)

## v1.8.0 (2026-07-12)

### 追加
- **起動場所の表示**: Windows版は一覧に「起動場所」列(レジストリキー/フォルダ/タスクパス/ストアアプリ)を
  追加し、ツールチップとCSVにも出力。macOS版は `i <番号>` で起動場所(plistパス等)を含む詳細を表示

### バグ修正(多エージェント・コードレビューで検出したデータ損失/誤動作)
Windows版:
- 新規追加ダイアログの「登録方法」と「対象」のラジオボタンが1グループになっており、対象を変えると
  登録方法が外れて意図と違う場所に登録されていた問題を修正(GroupBoxで独立した2グループに分離)
- 完全削除のたびに新しいバックアップを作成するよう修正(以前はセッション最初の1回のみで、
  2件目以降の削除でレジストリ項目が復元不能だった)
- 削除したタスクを復元すると元のフォルダ位置(TaskPath)が失われ、ルート直下に復元されていた問題を修正
  (`.meta` に元の TaskName/TaskPath を保存し、正確に復元)
- 同名タスクを続けて削除するとバックアップXMLが上書きされ片方が失われる問題を修正(ファイル名を一意化)
- 管理者権限なしでHKLM項目を削除しようとすると、実際は失敗しているのに「削除しました」と表示していた
  問題を修正(エラーを握り潰さず、失敗として集計・表示)
- `-SelfTest` が実在ストアアプリの状態を書き換えていた問題を修正(専用ダミーキーのみで検証)

macOS版:
- 削除の直前、バックアップのコピー失敗(root所有plist等)を握り潰したまま `sudo rm` していた問題を修正
  (バックアップの実在を確認できた項目だけ削除。root所有plistは `sudo cp` で退避)
- `d`/`e`/`r`/`s` の後に一覧を再表示するよう修正(番号がずれて別項目を操作する事故を防止)
- ログイン項目の削除で、名前に引用符/バックスラッシュを含むと失敗・任意コード実行の恐れがあった問題を修正
  (名前を AppleScript の argv 経由で安全に渡す)
- 無効/有効の判定で、ラベルに `.` `+` `(` 等を含むと誤判定していた問題を修正(固定文字列一致に変更)
- ログイン項目は復元不可であることを削除時とREADMEに明示

### 改良
- macOS版: 対話コマンド d/e の後処理を r と同じ安全な形に統一
- セルフテストを拡充(Windows: 起動場所・ダミーUWP状態切替 / macOS: CSVエスケープ・メタ文字ラベル判定)
